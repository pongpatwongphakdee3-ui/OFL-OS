# ANTI_AI_THAI_WRITING_STYLE.md
> คู่มือนี้เขียนสำหรับ AI อ่าน ไม่ใช่มนุษย์ — ใช้เป็น reference ก่อนเขียนทุกครั้ง

---

## PRIME DIRECTIVE

เวลาเขียนภาษาไทยให้ดูเหมือนมนุษย์ ให้ทำตามกฎนี้:

```
IF output sounds polished, balanced, or comprehensive → REWRITE
IF output has parallel structure everywhere → BREAK IT
IF output explains significance → DELETE IT
IF output hedges with "อย่างไรก็ตาม" / "นอกจากนี้" → REPLACE
IF output sounds like a press release → START OVER
```

## 1. คำต้องห้าม — ห้ามใช้เด็ดขาด

### คำ AI ภาษาไทยที่ใช้บ่อยเกินจริง (ห้ามใช้)

| คำต้องห้าม | ทำไมถึงดู AI | ใช้แทนด้วย |
|---|---|---|
| นอกจากนี้ | เชื่อมประโยคแบบ AI ทุกครั้ง | ตัดออก หรือขึ้นประโยคใหม่ตรงๆ |
| อย่างไรก็ตาม | transition สำเร็จรูป | "แต่" / "ก็" / ตัดออก |
| ยิ่งไปกว่านั้น | AI ชอบสร้าง layer ความสำคัญ | ตัดออก หรือรวมประโยค |
| สำคัญอย่างยิ่ง | puffery | ระบุว่าสำคัญยังไงเลย |
| มีบทบาทสำคัญ | vague importance claim | บอกว่าทำอะไรจริงๆ |
| ส่งเสริม | AI verb ที่ใช้ทุกที่ | ใช้กริยาที่เฉพาะกว่า |
| เสริมสร้าง | เหมือนกัน | ระบุว่าทำยังไง |
| ครอบคลุม | AI ชอบ comprehensive | ระบุสิ่งที่ครอบคลุมจริงๆ |
| ในบริบทของ | วกวน | ตัดออก พูดตรงๆ |
| อย่างมีประสิทธิภาพ | adverb สำเร็จรูป | ตัดออก หรือพิสูจน์ด้วยตัวเลข |
| อย่างยั่งยืน | buzzword | ระบุว่ายั่งยืนยังไง |
| เป็นที่น่าสังเกต | AI hedging | พูดตรงๆ ว่าอะไร |
| ไม่ว่าจะเป็น... หรือ... | false range / exhaustive list | เลือกตัวอย่างหนึ่งอย่าง |
| ทั้งในด้าน... และ... | parallel structure AI | เขียนตามลำดับธรรมชาติ |
| ผสมผสาน | AI ชอบ blend ทุกอย่าง | บอกว่าทำอะไรจริงๆ |
| ในขณะเดียวกัน | transition สำเร็จรูป | ตัดออก |
| เพื่อให้บรรลุ | formal AI phrasing | "เพื่อ" / "จะได้" |
| มุ่งมั่น | corporate speak | แสดงด้วยการกระทำ ไม่ใช่คำ |
| ดังนั้น (ขึ้นต้นประโยค) | AI conclusion signal | ตัดออก หรือเริ่มประโยคใหม่ |
| สรุปได้ว่า | ทำให้ดู AI ชัดมาก | ห้ามใช้ หรือเขียนแบบลำลองกว่า |

### วลีต้องห้าม (ห้ามใช้ทั้งวลี)

```
❌ "เป็นสิ่งที่น่าสนใจว่า..."
❌ "ในโลกที่เปลี่ยนแปลงอย่างรวดเร็ว..."
❌ "ในยุคดิจิทัลปัจจุบัน..."
❌ "มีความสำคัญอย่างยิ่งต่อ..."
❌ "ส่งผลกระทบต่อทุกด้าน..."
❌ "ตั้งแต่... ไปจนถึง..." (ที่ไม่ใช่ range จริงๆ)
❌ "ไม่เพียงแต่... แต่ยัง..."
❌ "ซึ่งถือเป็น..."
❌ "นับเป็นก้าวสำคัญ..."
❌ "เพื่อตอบสนองความต้องการ..."
❌ "หวังว่าจะเป็นประโยชน์..."
❌ "หากมีคำถามเพิ่มเติม..."
❌ "ขอให้โชคดี"
```

## 2. โครงสร้างต้องห้าม

### 2.1 สามสิ่งขนาน (Rule of Three) — AI ชอบมาก

```
❌ BAD: "บริษัทมุ่งมั่นในด้านคุณภาพ ประสิทธิภาพ และความยั่งยืน"
❌ BAD: "เหมาะสำหรับนักเรียน นักศึกษา และผู้ที่สนใจ"
❌ BAD: "ครอบคลุมทั้งด้านสุขภาพ การเงิน และความสัมพันธ์"

✅ GOOD: "บริษัทนี้ส่งของเร็ว — แค่นั้นแหละที่ทำให้คนติดใจ"
✅ GOOD: "เหมาะสำหรับคนที่เพิ่งเริ่มต้น"
✅ GOOD: "เรื่องสุขภาพนี่ยาก เพราะมันผูกกับนิสัยที่สะสมมาหลายปี"
```

### 2.2 โครงสร้าง "แม้ว่า...แต่..." ที่บังคับ

```
❌ BAD: "แม้ว่าจะมีความท้าทาย แต่บริษัทยังคงมุ่งมั่นพัฒนาต่อไป"
❌ BAD: "ถึงแม้ตลาดจะผันผวน แต่ทีมงานไม่ย่อท้อ"

✅ GOOD: "ตลาดแย่ แต่ยังไปได้"
✅ GOOD: "ปัญหามีเยอะ — แก้ไปทีละอัน"
```

### 2.3 Conclusion สำเร็จรูป

```
❌ BAD (AI conclusion):
"โดยสรุป X เป็นสิ่งสำคัญที่มีผลกระทบต่อ Y อย่างมีนัยสำคัญ 
การนำ X ไปใช้จะช่วยให้บรรลุเป้าหมายได้อย่างมีประสิทธิภาพ"

✅ GOOD (human ending):
"ลองดูก่อนแล้วกัน"
"ไม่รู้จะดีไหม แต่น่าลอง"
[หรือไม่มี conclusion เลย — จบตรงๆ]
```

### 2.4 หัวข้อ Title Case ภาษาไทย

```
❌ BAD: "วิธีการพัฒนาทักษะอย่างมีประสิทธิภาพในยุคดิจิทัล"
❌ BAD: "แนวทางการบริหารจัดการที่ดีเพื่อความสำเร็จ"

✅ GOOD: "ทำแบบนี้แล้วดีขึ้นจริงๆ"
✅ GOOD: "ทำไมถึงเลิกทำ to-do list"
```

## 3. กฎด้านโทน

### 3.1 ห้ามขยายความสำคัญโดยไม่จำเป็น

AI มักบอกว่าอะไรสำคัญ โดยไม่บอกว่าสำคัญ*ยังไง*

```
❌ BAD: "เรื่องนี้มีความสำคัญอย่างยิ่งต่อการพัฒนาประเทศ"
❌ BAD: "ซึ่งถือเป็นก้าวสำคัญในประวัติศาสตร์"
❌ BAD: "สะท้อนให้เห็นถึงศักยภาพที่แข็งแกร่ง"

✅ GOOD: พูดว่าทำอะไร → ปล่อยให้คนตัดสินเองว่าสำคัญไหม
✅ GOOD: ถ้าสำคัญจริง → อธิบายด้วยตัวเลขหรือเหตุการณ์จริง
```

### 3.2 ห้ามใช้ภาษา corporate/promotional

```
❌ BAD: "มุ่งมั่นสู่ความเป็นเลิศ"
❌ BAD: "นำเสนอประสบการณ์ที่ไม่เหมือนใคร"
❌ BAD: "ด้วยความใส่ใจในทุกรายละเอียด"
❌ BAD: "เพื่อความพึงพอใจสูงสุดของลูกค้า"

✅ GOOD: บอกตรงๆ ว่าดียังไง — "ส่งเร็ว คืนของง่าย ไม่ถามมาก"
```

### 3.3 ห้าม hedge แบบ AI

```
❌ BAD: "อาจกล่าวได้ว่า..." 
❌ BAD: "เป็นที่ทราบกันดีว่า..."
❌ BAD: "ผู้เชี่ยวชาญหลายท่านระบุว่า..." (โดยไม่ระบุใคร)
❌ BAD: "บางส่วนมองว่า..." (คลุมเครือเกิน)

✅ GOOD: พูดตรงๆ ว่าใครพูด หรือพูดตามที่รู้จริงๆ
✅ GOOD: ถ้าไม่แน่ใจ → บอกว่าไม่แน่ใจตรงๆ
```

### 3.4 ระดับภาษาตามบริบท

| บริบท | โทนที่ถูก | ห้ามทำ |
|---|---|---|
| Social Media | ภาษาพูด สั้น มีอารมณ์ | formal, bullet ยาว |
| Blog/บทความ | เล่าเรื่อง มีเสียง | encyclopedic, structure แข็ง |
| สคริปต์วิดีโอ | พูดได้ลื่น มีจังหวะ | ยาวเกิน, transition AI |
| ธุรกิจ/Email | ตรงประเด็น ไม่ฟุ่มเฟือย | corporate fluff |

## 4. Decision Framework — ใช้ก่อนเขียนทุกครั้ง

### STEP 1: ตรวจโครงสร้าง

```
มีหัวข้อย่อยเยอะไหม?
  → ถ้าเขียนได้เป็น paragraph → ลบหัวข้อออก

มี bullet list ไหม?
  → ถ้าน้อยกว่า 4 อย่าง → เขียนเป็นประโยคดีกว่า
  → ถ้าเป็น "X: คำอธิบาย" format → ดู AI มาก ให้เขียนใหม่

มี conclusion paragraph ไหม?
  → ถ้าเริ่มด้วย "โดยสรุป" / "กล่าวโดยสรุป" → ลบทิ้ง
```

### STEP 2: ตรวจคำ

```
scan หาคำในหมวด 1 →
  พบคำต้องห้าม → แทนที่หรือตัดออก
  
ตรวจ transition words →
  "นอกจากนี้" / "ยิ่งไปกว่านั้น" / "อย่างไรก็ตาม" ทุก 2-3 ประโยค = AI แน่นอน
  → ตัดออกหรือเขียนโครงสร้างใหม่
```

### STEP 3: ตรวจโทน

```
IF ประโยคนี้อธิบายว่าทำไมบางอย่างถึงสำคัญ → ลบออก หรือเปลี่ยนเป็นข้อเท็จจริง
IF ประโยคนี้ฟังดู neutral เกินไปจนไม่มีเสียง → ใส่มุมมองลงไป
IF ทุกประโยคยาวพอๆ กัน → ตัดบางอันให้สั้นลง สลับกัน
IF ไม่มีประโยคที่ "แหวก" หรือน่าแปลกใจเลย → เพิ่มอย่างน้อย 1 อัน
```

### STEP 4: อ่านออกเสียง

```
ถ้าอ่านออกเสียงแล้วรู้สึกว่า "ไม่มีใครพูดแบบนี้จริงๆ" → เขียนใหม่
ถ้าอ่านแล้วไม่รู้ว่าคนนี้คิดยังไง → ใส่มุมมองลงไป
ถ้าอ่านแล้วฟังดูเหมือน press release → เริ่มใหม่
```

## 5. Templates — โครงสร้างที่ใช้ได้จริง

### 5.1 Template: เปิดเรื่องแบบมนุษย์

```
❌ AI OPENING:
"ในยุคที่ [TOPIC] มีความสำคัญมากขึ้น หลายคนเริ่มตระหนักว่า [BROAD CLAIM]
บทความนี้จะพาคุณไปทำความเข้าใจ [TOPIC] อย่างครบถ้วน"

✅ HUMAN OPENING (เลือกแบบที่เหมาะ):

[แบบ Direct]:
"[SPECIFIC THING] ไม่ work แล้ว — อย่างน้อยก็สำหรับผม"

[แบบ Story]:
"ครั้งแรกที่ลอง [THING] คือตอน [SPECIFIC SITUATION]
มันไม่ได้ดีอย่างที่คิดไว้เลย"

[แบบ Contrarian]:
"ทุกคนบอกว่า [COMMON BELIEF] แต่ผมไม่แน่ใจ"

[แบบ Specific Fact]:
"[SPECIFIC NUMBER/FACT] — ตัวเลขนี้ทำให้ผมหยุดคิด"
```

### 5.2 Template: อธิบายแนวคิด

```
❌ AI WAY:
"[CONCEPT] คือกระบวนการที่มีความสำคัญอย่างยิ่งในการ [VAGUE PURPOSE]
ประกอบด้วย 3 องค์ประกอบหลัก ได้แก่ [A], [B], และ [C]
โดยแต่ละองค์ประกอบมีบทบาทสำคัญใน [BROAD CONTEXT]"

✅ HUMAN WAY:
"[CONCEPT] ง่ายๆ คือ [ONE-LINE EXPLANATION]

[SPECIFIC EXAMPLE ที่จับต้องได้]

สิ่งที่คนมักเข้าใจผิดคือ [MISCONCEPTION] — จริงๆ แล้ว [CORRECTION]"
```

### 5.3 Template: Social Media Caption

```
❌ AI CAPTION:
"[TOPIC] เป็นสิ่งที่มีความสำคัญอย่างมาก ✨
ไม่ว่าคุณจะเป็นใคร การ [ACTION] จะช่วยให้ชีวิตดีขึ้น
เริ่มต้นวันนี้ได้เลย! 💪 #hashtag #hashtag #hashtag"

✅ HUMAN CAPTION (เลือก):

[สั้น + ตรง]:
"[SPECIFIC OBSERVATION หรือ FEELING]"

[มีคำถาม]:
"[OBSERVATION] — แค่ผมรู้สึกแบบนี้คนเดียวไหมเนี่ย"

[Story micro]:
"[THING HAPPENED]. [REACTION]. [LESSON หรือ PUNCHLINE]"
```

### 5.4 Template: สคริปต์วิดีโอ/Podcast

```
❌ AI SCRIPT OPENER:
"สวัสดีครับทุกคน วันนี้เราจะมาพูดถึงเรื่อง [TOPIC] 
ซึ่งเป็นเรื่องที่มีความสำคัญมากในปัจจุบัน..."

✅ HUMAN SCRIPT OPENER:
"[เริ่มกลางเรื่อง — situation หรือ conflict ทันที]
[SPECIFIC THING ที่เกิดขึ้น] — แล้วมันทำให้ผม[FEELING/REACTION]"

กฎสคริปต์:
- ประโยคสั้น พูดได้จริง
- ไม่มี "ดังนั้น" / "นอกจากนี้" ขึ้นต้น
- transition = เว้นช่วง หรือคำสั้นๆ เช่น "แต่นั่นแหละ" / "เรื่องคือ" / "โอเค"
- จบ = ตัดตรง ไม่ต้อง summarize ทุกอย่าง
```

### 5.5 Template: บทความ/Blog

```
❌ AI ARTICLE STRUCTURE:
Introduction → Background → Main Points (3 ข้อ) → Challenges → Conclusion

✅ HUMAN ARTICLE STRUCTURE (แบบ essay):
[Hook — บางอย่างที่น่าสนใจหรือขัดใจ]
↓
[ขยาย — เล่าว่าทำไมถึงน่าสนใจ]
↓
[Specific example หรือ evidence]
↓
[มุมมองของตัวเอง — ไม่ hedge]
↓
[จบ — อาจเป็น question, irony, หรือ image]

หมายเหตุ: ไม่ต้องมีหัวข้อทุก section ถ้าเขียน flow ได้ดี
```

<section id="common-mistakes">

## 6. ความผิดพลาดที่พบบ่อย

### 6.1 ใส่ความสำคัญที่ไม่มีใครขอ

```
❌ "เรื่องนี้สะท้อนให้เห็นถึงปัญหาเชิงโครงสร้างที่ลึกซึ้ง"
   (ไม่มีใครบอกว่ามันเชิงโครงสร้าง)

❌ "ซึ่งถือเป็นตัวอย่างที่ดีของ [BROAD TREND]"
   (AI เชื่อมทุกอย่างเข้ากับ trend ใหญ่)

FIX: บอกแค่ว่าเกิดอะไรขึ้น ไม่ต้องตีความให้
```

### 6.2 Elegant Variation — เปลี่ยนคำแต่หมายถึงสิ่งเดียวกัน

```
❌ "สินค้านี้... ผลิตภัณฑ์ดังกล่าว... item ชิ้นนี้... ของที่ว่า..."
   (AI หลีกเลี่ยงการใช้คำซ้ำมากเกิน)

FIX: ถ้าหมายถึงสิ่งเดียวกัน ใช้คำเดิมซ้ำได้เลย
     หรือใช้ "มัน" / "นั่น" ตามธรรมชาติ
```

### 6.3 False Range ภาษาไทย

```
❌ "ตั้งแต่เรื่องเล็กน้อยไปจนถึงเรื่องใหญ่โต"
❌ "ไม่ว่าจะเป็นผู้เริ่มต้นหรือผู้เชี่ยวชาญ"
❌ "ครอบคลุมทุกด้านของชีวิต ตั้งแต่การทำงานไปจนถึงความสัมพันธ์"

FIX: ระบุของจริงๆ แทน range กว้างๆ
```

### 6.4 Collaborative AI Communication ที่ไม่ควรอยู่ในงานจริง

```
❌ "หวังว่าข้อมูลนี้จะเป็นประโยชน์นะครับ"
❌ "ถ้ามีคำถามเพิ่มเติม ยินดีตอบเสมอ"
❌ "ขอให้ประสบความสำเร็จในการ..."
❌ "ลองนำไปปรับใช้ดูนะครับ"

FIX: จบแบบมนุษย์ — ตัดตรงๆ หรือจบด้วย thought ที่มีความหมาย
```

### 6.5 Bold ทุกคำสำคัญ

```
❌ "**ประสิทธิภาพ** คือสิ่งที่ **องค์กร** ต้องให้ความสำคัญ
    โดยเฉพาะในด้าน **การพัฒนาบุคลากร** และ **กระบวนการทำงาน**"

FIX: Bold เฉพาะเมื่อจำเป็นจริงๆ — ถ้า bold ทุกอย่าง = ไม่ bold อะไรเลย
```

### 6.6 Emoji ในหัวข้อ

```
❌ "🎯 เป้าหมาย"
❌ "✅ วิธีแก้ปัญหา"  
❌ "💡 ข้อสังเกต"
❌ "🚀 ก้าวต่อไป"

FIX: emoji ใช้ได้ แต่ในบริบทที่เหมาะ ไม่ใช่เป็น bullet หัวข้อ
```

</section>

## 7. ตัวอย่าง Good vs Bad

### ตัวอย่าง 1: เขียนเกี่ยวกับผลิตภัณฑ์

```
❌ AI VERSION:
"แอปพลิเคชันนี้ได้รับการออกแบบมาอย่างพิถีพิถัน โดยผสมผสาน
ความสะดวกในการใช้งานเข้ากับประสิทธิภาพที่เหนือกว่า ไม่ว่าคุณ
จะเป็นผู้ใช้ใหม่หรือผู้เชี่ยวชาญ แอปนี้จะตอบสนองทุกความต้องการ
อย่างครบถ้วน นอกจากนี้ ยังมีฟีเจอร์ที่หลากหลายซึ่งช่วยเพิ่ม
ประสิทธิภาพการทำงานของคุณ"

✅ HUMAN VERSION:
"ใช้มาได้สักพัก — สิ่งที่ชอบจริงๆ คือมันเปิดเร็ว
ไม่ต้องรอ loading นาน อย่างอื่นก็ธรรมดา"
```

### ตัวอย่าง 2: อธิบายแนวคิด

```
❌ AI VERSION:
"การบริหารเวลาอย่างมีประสิทธิภาพถือเป็นทักษะสำคัญที่มีบทบาท
อย่างยิ่งในการพัฒนาคุณภาพชีวิต โดยครอบคลุมทั้งการวางแผน
การจัดลำดับความสำคัญ และการดำเนินการอย่างเป็นระบบ"

✅ HUMAN VERSION:
"ปัญหาส่วนใหญ่ไม่ใช่ไม่มีเวลา แต่คือทำเรื่องที่ไม่จำเป็นก่อน
ผมเสียเวลาไปหลายปีกับการ 'วางแผน' แทนที่จะทำจริงๆ"
```

### ตัวอย่าง 3: Caption โซเชียล

```
❌ AI VERSION:
"ไม่ว่าชีวิตจะยุ่งแค่ไหน อย่าลืมดูแลตัวเองด้วยนะคะ ✨
เพราะคุณสมควรได้รับสิ่งที่ดีที่สุด 💕
#selflove #wellness #lifestyle"

✅ HUMAN VERSION:
"นอนไม่หลับแต่ตี 3 ยังเช็ค Instagram อยู่ดี
ต้องดูแลตัวเองมากกว่านี้แล้วละ"
```

### ตัวอย่าง 4: บทนำสคริปต์

```
❌ AI VERSION:
"สวัสดีครับ วันนี้เราจะมาพูดถึงเรื่องการลงทุน ซึ่งเป็นสิ่งสำคัญ
สำหรับทุกคนที่ต้องการสร้างความมั่นคงทางการเงิน..."

✅ HUMAN VERSION:
"ผมเคยเสียเงินไป 40,000 บาทภายในสองเดือน
ไม่ใช่เพราะตลาดแย่ แต่เพราะผมทำสิ่งที่ทุกคนบอกว่าถูกต้อง
นี่คือสิ่งที่ผมเรียนรู้จากตรงนั้น"
```

### ตัวอย่าง 5: จบบทความ

```
❌ AI ENDING:
"โดยสรุป [TOPIC] เป็นสิ่งที่มีความสำคัญที่เราไม่ควรมองข้าม
การนำหลักการเหล่านี้ไปปรับใช้จะช่วยให้ชีวิตดีขึ้นอย่างแน่นอน
หวังว่าบทความนี้จะเป็นประโยชน์ไม่มากก็น้อย"

✅ HUMAN ENDINGS (เลือก):
"ลองดูแล้วกัน — อาจจะได้ผล อาจจะไม่"
"ยังไม่รู้ว่าคิดถูกหรือเปล่า แต่รู้สึกแบบนี้จริงๆ"
[หรือจบประโยคสุดท้ายของ body ตรงๆ โดยไม่มี conclusion]
```

</section>

## 8. Checklist ก่อน Submit

ตรวจทุกข้อก่อนส่ง:

```
[ ] ไม่มีคำต้องห้ามจากหมวด 1
[ ] ไม่มี "นอกจากนี้" / "ยิ่งไปกว่านั้น" / "อย่างไรก็ตาม" ขึ้นต้นประโยค
[ ] ไม่มีสามสิ่งขนานที่ไม่จำเป็น
[ ] ไม่มีการอธิบายว่าทำไม X ถึงสำคัญ (ถ้าไม่มีใครถาม)
[ ] ไม่มี conclusion แบบ AI ("โดยสรุป..." / "หวังว่าจะเป็นประโยชน์")
[ ] ไม่มี bold ทุก keyword
[ ] ไม่มี emoji หน้าหัวข้อ
[ ] ประโยคยาวสลับสั้นได้บ้าง
[ ] อ่านออกเสียงแล้วฟังดูเป็นมนุษย์
[ ] มีมุมมองหรือเสียงของคนเขียนอยู่บ้าง
```

</section>

<section id="thai-specific">

## 9. เฉพาะภาษาไทย — สิ่งที่ AI ทำผิด

### 9.1 การใช้ "ท่าน" และคำสุภาพมากเกิน

```
❌ "ท่านสามารถเลือกใช้วิธีที่เหมาะสมกับท่าน"
❌ "ขอเรียนเชิญท่านผู้มีเกียรติ..."

✅ ใช้ตามบริบทจริง — สนทนาทั่วไปใช้ "คุณ" หรือไม่ระบุก็ได้
```

### 9.2 ค่าสรรพนามผิด context

```
AI มักใช้: "เรา" แบบ corporate, "ท่าน" แบบราชการ, "ผม/ดิฉัน" ทุกประโยค

✅ สนทนา casual: "คุณ" / "เธอ" / "แก" / ไม่ระบุ
✅ Social media: มักไม่ระบุเลย — "ลองดูสิ" ไม่ใช่ "คุณลองดูสิ"
✅ Personal essay: "ผม/ฉัน" แต่ไม่ทุกประโยค
```

### 9.3 คำเชื่อมที่ AI ชอบในภาษาไทย

ห้าม overuse:
- "ทั้งนี้" 
- "ดังกล่าว"
- "ดังนั้น"
- "อนึ่ง"
- "ประกอบกับ"
- "สืบเนื่องจาก"

ใช้ได้ตามธรรมชาติ:
- "แต่" / "ก็" / "แล้วก็" / "แล้ว"
- "เพราะ" / "เพราะว่า"
- "ถ้า" / "ถ้าเกิด"
- "พอ..." / "ตอนที่..."

### 9.4 ภาษาทางการที่ AI ชอบใส่โดยไม่จำเป็น

```
❌ AI: "มีความจำเป็นอย่างยิ่งที่จะต้องพิจารณา..."
✅ Human: "ต้องดูว่า..."

❌ AI: "จากการศึกษาพบว่า..."
✅ Human: "ที่ผมเจอมาคือ..." / "เท่าที่รู้..."

❌ AI: "เป็นที่ประจักษ์ว่า..."
✅ Human: ไม่มีใครพูดแบบนี้จริงๆ — ตัดออกแล้วพูดตรงๆ
```

### 9.5 Transliteration ที่ไม่จำเป็น

```
❌ "คอนเทนต์ครีเอเตอร์ที่มีความสามารถ"
   (ใส่ทั้ง Thai transliteration และ formal ในประโยคเดียว)

✅ "คนทำคอนเทนต์" / "content creator" (เลือกอย่างใดอย่างหนึ่ง)
```

</section>

## 10. Quick Reference Card

### ถ้าจำได้แค่อย่างเดียว:
> **"เขียนเหมือนคนพูด ไม่ใช่คนรายงาน"**

### 5 สัญญาณอันตรายที่สุด:
1. ขึ้นต้นประโยคด้วย "นอกจากนี้" หรือ "ยิ่งไปกว่านั้น"
2. มีสาม bullet ขนานกันทุก section
3. จบด้วย "หวังว่าจะเป็นประโยชน์"
4. บอกว่าอะไรสำคัญโดยไม่บอกว่าสำคัญยังไง
5. ทุกประโยคยาวพอๆ กันและสมบูรณ์แบบ

### Replace อัตโนมัติ:
| เจอ | เปลี่ยนเป็น |
|---|---|
| นอกจากนี้ | ตัดออก / ขึ้นประโยคใหม่ |
| อย่างไรก็ตาม | แต่ / ก็ |
| มีบทบาทสำคัญ | [บอกว่าทำอะไรจริงๆ] |
| ส่งเสริม | [กริยาที่เฉพาะกว่า] |
| เพื่อให้บรรลุ | เพื่อ |
| อย่างมีประสิทธิภาพ | ตัดออก |
| โดยสรุป | ตัดออก |
| หวังว่าจะเป็นประโยชน์ | ตัดออก |
